the problem

  • procurement data comes from all kinds of different sources, each with their own set of unique challenges
    • dated legacy systems vs. state of the art data management
    • cleanliness of manual entries
    • massive storage and memory demands
    • and much more...

  • furthermore, these sources are often highly connected
    • these connections are critical for comprehensive analysis, but are often missed due to the complexity

  • this is where Rotunda Solutions comes in!

context: data collection

  • even given a relatively small scope, analysis is a very difficult task to do correctly

  • some considerations include:
    • what data sources should one start with?
    • how can one access and store the data from each?

  • for example, one may miss fine details different (and often hard to access) sources bring:
    • FPDS-NG RSS feed: contract data
    • SAM REST API: vendor data
    • USA Spending API: subawardee data to complement vendor data

context: data modeling

  • even when using the correct sources and pulling the data, there are many tricks to maximize insights
    • example: using knowledge graphs to highlight 'hidden' connections between disparate sources

  • consider the real-world mini-graph on the right, detailing relations between:
    • vendors indexed by DUNS codes
    • contracts the vendors were awarded with

  • this gives us some useful information, but how could we dig even deeper?

supercharged insights

  • many would stop here, but we can often mine more information by tracing 'deeper' connections in the graph
    • here we have added company ownership links; note how many more vendors have been linked to these contracts!

  • we now have a more complete landscape, which could be critical in multiple analysis, including:
    • analyzing competing vendors
    • ensuring conglomerates do not box out small business from protected opportunities

Vendor Performance Location Map

  • motivation: where are vendors in this scope working?
    • how much is each vendor making relative to the others?
    • do any locations seem to be general hotspots for high award totals?

  • visualization guide:
    • points represent vendors in this scope
    • vendors are colored by (scaled) total earnings

NAICS Breakdown

  • motivation: what contracts/vendors/locations are associated with NAICS code 'X'?
    • are any locations linked to all NAICS codes?
    • are any NAICS codes very specific to a single vendor/location?

  • visualization guide:
    • node types are represented by columns
    • links between nodes reflect per real-world connections

Vendor Specialty Drilldown

  • motivation: within a defined scope of contracts, what do vendors specialize in?
    • do any vendors have a monopoly over a specific kind of award?
    • do any NAICS codes disprorortionately account for total obligation costs?

  • visualization guide:
    • contracts in the scope are grouped by NAICS codes and prime vendor DUNS
    • cells are sized by contract award amount

conflict of interest detection

  • given a company (e.g. "Rotunda Solutions"), how can we identify as many currently active and connected programs that the company was a prime and/or subcontractor on?

  • example: Sam was promoted to Defense Secretary. How can he get ahead of any potential accusations of conflicts of interest with Rotunda?
    • imagine if the 'QWERTY' weapons program awarded tons of work to Rotunda...
    • what if a subsidary 'Adnutor Solutions' was associated with the program; how could an NLP solution catch this?

a creative solution

  • let's break this project down into three main pieces:
    1. data merging
    2. data modeling
    3. querying for results

  • now that we have the overall project context down, we can go over some more fine details

leveraging connections

  • now let's focus on the data modeling part

  • this is actually much simpler than it may sound at first:

    1. build up structures that record all attributes associated with each node

    2. link the nodes together (using NetworkX, Neo4j or some other technology)

    3. chain data attributes to answer meanginful analysis questions
